## Status Definitions

| Status | Definition |
|--------|-----------|
| **Open** | Assigned but not yet started |
| **In Progress** | Work has begun |
| **Complete** | Action fulfilled, verified |
| **Cancelled** | No longer required |
| **Overdue** | Past due date, not yet complete |

---

## Priority Definitions

| Priority | Definition | Expected Response |
|----------|-----------|-------------------|
| **High** | Critical to project progress, blocking other work | Complete within 2 business days |
| **Medium** | Important but not blocking | Complete by assigned due date |
| **Low** | Desirable but not urgent | Complete when capacity allows |

## Instructions

1. **Creating:** Assign a unique sequential ID (AI-XXX) when creating new items
2. **Ownership:** Every action item must have a single named owner
3. **Due dates:** All items must have a due date — negotiate if needed at time of assignment
4. **Updates:** Owners update status at each team meeting or when status changes
5. **Closure:** Mark complete only when the action is verified as done
6. **Review:** Review all open/overdue items at each status meeting
