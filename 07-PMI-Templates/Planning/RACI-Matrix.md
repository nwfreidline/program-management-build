## Instructions for Use

### Rules
1. **Every activity must have exactly one "A"** — only one person is accountable
2. **Every activity must have at least one "R"** — someone must do the work
3. **The same person can be both R and A** for a given activity
4. **Minimize the number of C's and I's** — too many slows decisions
5. **Review with all named stakeholders** before finalizing

### Common Issues to Watch For
- **No A assigned** — no clear ownership leads to dropped balls
- **Multiple A's** — diffused accountability; only one person can be accountable
- **Too many R's** — unclear who is actually doing the work
- **Too many C's** — decision bottlenecks; consult only those truly needed
- **Empty rows** — an activity with no assignments needs attention
- **Overloaded columns** — one person with too many R's may be a bottleneck

### Validation Checklist
- [ ] Every row has exactly one A
- [ ] Every row has at least one R
- [ ] No person is overloaded across all activities
- [ ] All stakeholders have reviewed and agreed to their assignments
- [ ] Matrix aligns with the WBS and project plan
